# Интерфейс ученика · AI Math AZ

Кликабельный прототип приложения ученика для проекта «Intelligent Math Classroom on Tablets» (Азербайджан, 4 класс). Парный к [прототипу учителя](https://github.com/yuvakhrusheva/teacher-prototype) — тот же класс (4-А, школа №297), тот же урок 84 «Деление с остатком».

**Спецификация экранов:** [SCREENS_SPEC.md](SCREENS_SPEC.md) — JTBD, обязательный функционал, открытые вопросы и источники данных по каждому экрану.

## Что внутри

Один файл `index.html` без зависимостей (шрифт Inter локально). Слева — навигация по экранам, в центре — макет устройства.

- **В классе (планшет, ландшафт):** S1 ожидание урока → S2 разминка → S3 объяснение (view-only) → S4 практика (трек A/B/C) → S5 фото тетради → S6 «не получается» → S7 рефлексия → S8 урок завершён + ДЗ.
- **Дома (мобильный, портрет):** S9 главная (активное ДЗ + формативная ОС «по итогам урока» + домашки в срок) → S10 домашка → S11 сдано/разбор → S12 мой прогресс (карта тем).
- **Заглушки:** S13 буферный урок, S14 диагностика.

Пунктирные кнопки над макетом — симуляция событий вне планшета (сигналы учителя: «начать урок», «перелистнуть слайд»).

## Ключевые принципы

1. Планшет управляется фазой урока — свободной навигации в классе нет; все переходы (включая рефлексию и завершение урока) запускает учитель.
2. Треки A/B/C ученику не показываются — адаптив работает «под капотом», ребёнок просто решает.
3. Без ачивок в MVP: ни стикеров, ни звёзд. Прогресс — статусы тем и счётчик «домашек сдано в срок»; пропущенные домашки (болезнь) можно «догнать».
4. Камера-флоу для письменных заданий: фото тетради → ночной Vision-разбор → утром у учителя.
