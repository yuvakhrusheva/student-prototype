# Экраны интерфейса ученика · подписи для Figma

Скриншоты в этой папке (ретина ×2, чистый экран без рамки устройства). Полная спецификация — в [`../SCREENS_SPEC.md`](../SCREENS_SPEC.md).
Сквозной контекст всех экранов: ученица **Айсель**, класс **4-А**, **урок 84 «Деление с остатком»** — тот же класс и урок, что в прототипе учителя.

## Принципы системы (общий контекст для слайда-обложки)
- Планшет в классе **управляется фазой урока** — все переходы (старт, объяснение, рефлексия, завершение) запускает учитель; свободной навигации в классе нет.
- Адаптив (треки A/B/C) работает **«под капотом»** — ученику уровень не показывается.
- **Без геймификации-ачивок** (нет очков, стикеров, стриков). Звёзды 1–3 — только рейтинг правильности домашки.
- Письменные задания фотографируются → ночной Vision-разбор → утром у учителя.

# Часть I · Планшет (в классе, ландшафт)

### `01-waiting.png` — Ожидание урока
Точка входа за партой: тема сегодняшнего урока, приветствие, статус «ждём учителя». Планшет авто-переходит в разминку по сигналу учителя.
*Источник: LMS (расписание) · сигнал старта от учителя.*

### `02-warmup.png` — Разминка (вопрос)
Все ученики решают одинаковые задания (3 подряд) с автопроверкой. Ответы в реальном времени уходят на экран учителя (распределение по вариантам).
*Источник: Движок задач · Планшет (live), детерминированная автопроверка.*

### `03-warmup-feedback.png` — Разминка (мгновенный фидбек)
После ответа — мгновенно «верно/неверно» с подсветкой правильного и переходом к следующему вопросу.

### `04-explanation.png` — Объяснение (view-only)
Планшет синхронизирован с проектором, ввод заблокирован («🔒 Смотрим вместе»). Слайды листает учитель.
*Источник: Движок задач (слайды) · режим синхронизации view-only.*

### `05-practice.png` — Практика (задача)
Сердце адаптива: после первых ответов система назначает трек, ученик решает свою последовательность. Статусы (решает/застрял/ошибка/опередил) уходят на live-карту учителя.
*Источник: Движок задач (банк по трекам) · Граф навыков (порог мастерства) · Планшет (live) + автопроверка (детерминированная + LLM на свободных).*

### `06-practice-written.png` — Практика (письменная задача)
Часть задач решается в тетради и фотографируется (до 5 фото/урок).

### `07-camera.png` — Фото тетради (камера)
Экран камеры с рамкой-подсказкой для съёмки страницы решения.
*Источник: Планшет (камера).*

### `08-camera-preview.png` — Фото тетради (превью)
Проверка снимка: переснять или отправить.

### `09-camera-sent.png` — Фото тетради (отправлено)
«Проверим к утру» — ночной Vision-разбор (CV+LLM, локализация шага ошибки) → утром результат у учителя.
*Источник: Vision (ночной разбор, привязка к критериям).*

### `10-stuck.png` — «Не получается» (тихий запрос помощи)
Тихий сигнал учителю: карточка ученика подсвечивается на его live-карте, никто из класса не видит. Закрывает боль учителя «точечная помощь вместо шума».
*Источник: Планшет (live) — статус на карту класса учителя.*

### `11-reflection.png` — Рефлексия
Короткая самооценка по эмодзи-шкале. Питает профиль ученика и сравнение «самооценка vs реальная решаемость» на экране учителя. Запускает фазу учитель.
*Источник: Рефлексия → профиль ученика.*

### `12-finished.png` — Урок завершён + ДЗ
Позитивная сводка урока (решено задач, верно с 1-й попытки) + карточка назначенного ДЗ (придёт в мобильный режим). Без ачивок.
*Источник: Планшет (агрегаты урока) · Движок задач + Граф навыков (состав ДЗ).*

# Часть II · Мобильный (дома, портрет)

### `13-home.png` — Домашняя главная
Хаб: активное ДЗ с дедлайном; **«По итогам урока»** — детская формативная ОС (похвала + 1 шаг роста, связка с ДЗ); счётчик «домашек сдано в срок» и пропущенные; вход в «Мой прогресс».
*Источник: Планшет/моб. (статус ДЗ) · LMS (дедлайны) · Движок ОС (формативная ОС, детская переформулировка) · Граф навыков.*

### `14-homework.png` — Домашка (решение)
Последовательное решение репродуктивных заданий по треку ученика с автопроверкой. Mobile-friendly: одно задание на экран.
*Источник: Движок задач (репродуктивный банк) · автопроверка.*

### `15-homework-done.png` — Домашка сдана
Результат + **звёзды 1–3 по правильности** + разбор ошибок. Опоздавшая домашка показывается так же (без ярлыка «с опозданием»).
*Источник: Планшет/моб. (ответы, автопроверка) · Граф навыков (привязка ошибок к критериям).*

### `16-progress.png` — Мой прогресс
**«Мои уроки»** — дорожка уроков со звёздами за домашку у каждого пройденного; отдельный блок **пропущенных домашек** (болезнь) с кнопкой «Догнать».
*Источник: Граф навыков (мастерство, фронтир) · LMS (структура программы) · Рефлексия.*

# Часть III · Заглушки (вне скоупа v1)

### `17-buffer.png` — Буферный урок (заглушка)
Раз в ~2 недели — персональный набор задач по треку (закрытие пробелов / база / сложные). Подаётся как «твой особенный урок» без ярлыка коррекции.
*Источник: Движок буферных уроков · Граф навыков.*

### `18-diagnostic.png` — Диагностика (заглушка)
3 раза в год — режим теста (симуляция мониторинга): без подсказок и без проверки до завершения.
*Источник: Диагностика.*
